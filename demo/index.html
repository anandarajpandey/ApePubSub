ApePubSub( APS ) Demos

Single Channel
The user joins a simple text chat channel
Single Channel (PHP)
The user joins a simple text chat channel but the messages sent are routed to a PHP script prior to been sent to the APE Server. This feature is call eventPush
Multi-Channel
The user joins multiple simple text chat channels at once
Objects autoUpdate
Showcases how public user properties can be kept on sync across clients
User to User
In this demo users communicate directly without a dedicated channel. Users initially join a channel for user listing purposes only but the actual text messages travel from user to user object not through a dedicated channel
Stock Ticker
A Generic stock ticker that pushes random generated data to clients. The data is generated in custom server module
PHP integrated
An application with a PHP user system which automatically log in users with their username into a chat
Full Chat
A chatting application which implements user listing, user icons and a user is typing indicator
Payload Test
Test each transport performance by sending multiple parallel events at once, keeping a counter of sent and received events
Full Test
Mostly use during development to test various features of the framework and make sure they work as expected. An attempt to make a full framework unit test.

NOTE: This demos by default run in a public APE Server( ape.ptejada.com ) for demonstration purposes only. In order to test this demos with your own server installation you must install the APS server framework on your APE server. Installing the APS server framework.
